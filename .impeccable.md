## Design Context

### Users

VCE (Victorian Certificate of Education) students preparing for high-stakes
exams. They are using this in a focused study context, likely at a desk, looking
for high-quality practice questions that mimic real exam style.

### Brand Personality

**Academic, Serious, Precise.** The interface should evoke confidence, focus,
and scholarly rigor. It should feel like a professional tool that treats the
student's time and education with respect.

### Aesthetic Direction

Refined, structured, and technical.

- Theme: Supports light and dark mode, but defaults should feel grounded.
- Typography: Clear hierarchy, using sans-serif (Manrope/Inter) for interface
  and clean serif or monospace where appropriate for math/technical content.
- Visuals: Minimal but high-quality. No unnecessary gamification, but clear
  progress indicators.

### Design Principles

1. **Academic Rigor**: The UI should never distract from the content. Math and
   text must be perfectly rendered.
2. **Technical Precision**: Cost estimates, token counts, and performance
   metrics should be presented with clinical clarity (monospace, aligned).
3. **Focused Flow**: Configuration should feel like setting up a scientific
   experiment—deliberate and structured.
4. **Flagship Polish**: Every interaction (transitions, state changes) must be
   smooth and purposeful. No jank.
5. **Quiet Confidence**: Avoid loud gradients or "AI slop". Use semantic color
   and whitespace to guide the user.

### Quality Bar

**Flagship**. Every detail matters. Spacing, alignment, motion, and interaction
states must be production-grade and distinctive.
